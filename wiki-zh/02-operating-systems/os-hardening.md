## 为什么这个主题重要

OS hardening，也就是操作系统加固，意思是减少操作系统上不必要的风险。它不是让电脑变成“永远不可能被攻破”。它是减少低级错误、限制攻击者能利用的东西，并且让系统更容易被监控。

初学者版本：

> 加固就是关掉不需要的，保护需要的，并确认我能发现问题。

## 基本想法

新装系统通常能用，但不一定足够安全。有些默认设置是为了方便，有些默认设置比较宽泛，因为操作系统要支持很多使用场景。

加固会问：

- 安装了什么软件？
- 哪些服务在运行？
- 有哪些账号？
- 谁有管理员权限？
- 更新是否安装？
- 防火墙是否开启？
- 日志是否可用？
- 敏感数据是否加密？
- 备份是否有效？

## 补丁

打补丁就是安装修复 bug 和安全漏洞的更新。

为什么重要：

- 已知漏洞通常更容易被利用。
- 攻击者可能扫描缺少旧补丁的系统。
- 安全工具不能完全弥补不打补丁的问题。

初学者练习：

- 检查更新状态。
- 在测试或安全窗口安装更新。
- 需要时重启。
- 记录改了什么。

安全提示：

有些补丁可能影响业务，所以组织会测试和安排时间。但永远不补丁更糟。

## 移除不需要的软件

每个安装的软件都会增加潜在风险。

例子：

- 旧浏览器
- 不用的远程访问工具
- 试用软件
- 不维护的小工具
- Downloads 里的未知安装包

初学者规则：

如果不知道某个程序为什么安装，先调查再删。但如果它明显不用也不必要，移除它能减少攻击面。

## 禁用不需要的服务

服务在后台运行。有些服务会监听网络端口。

为什么重要：

- 服务可能有漏洞。
- 服务可能暴露端口。
- 服务可能以高权限运行。

需要检查的例子：

- Remote Desktop
- SSH
- 文件共享
- 打印服务
- Web server
- 数据库服务

初学者提醒：

不要在生产系统上随机禁用服务。先在实验室里学。

## 账号加固

账号是最大安全区域之一。

需要检查：

- 不用的账号是否禁用？
- 谁是管理员？
- 默认账号是否仍启用？
- 密码是否强？
- 是否有 MFA？
- 服务账号权限是否过大？

安全角度：

- 管理员账号应该被限制。
- 日常工作不应该一直用管理员。
- 共享账号会让审计变困难。
- 老账号容易被忘记，也容易被滥用。

## 密码和锁定策略

有用控制：

- 最小密码长度
- 多次失败登录后的账号锁定
- 使用密码管理器
- 尽量开启 MFA

初学者笔记：

密码复杂度不是全部。长且唯一的密码，加上 MFA，通常比短但符号很多的密码更重要。

## 防火墙

主机防火墙控制机器上的网络流量。

基本目标：

- 阻止意外入站流量。
- 只允许需要的服务。
- 不随便暴露管理服务。

例子：

- Windows Firewall
- Ubuntu 上的 `ufw`
- 一些 Linux 系统上的 `firewalld`
- macOS Application Firewall

安全角度：

如果一个服务不需要被网络访问，防火墙就不应该允许它。

## 日志

日志帮助我理解发生了什么。

重要日志：

- 登录尝试
- 管理员操作
- 服务失败
- 安全告警
- 防火墙事件
- 应用错误

初学者错误：

只在坏事发生后才想起来看日志，然后发现日志没开，或者早就被覆盖了。

## 备份

备份也是安全的一部分，因为它能帮助从这些问题中恢复：

- 勒索软件
- 误删除
- 硬盘故障
- 糟糕更新
- 配置错误

好的备份问题：

- 备份了什么？
- 多久备份一次？
- 存在哪里？
- 谁能访问？
- 是否测试过恢复？

初学者笔记：

从没恢复测试过的备份，只是希望，不是证明。

## 加密

磁盘加密能在设备丢失或被偷时保护数据。

例子：

- Windows 上的 BitLocker
- macOS 上的 FileVault
- Linux 上的 LUKS

安全角度：

加密在攻击者拿到设备但没有解锁密钥时有帮助。系统已经解锁运行时，它不能保护所有东西。

## 最小权限

最小权限表示用户和程序只应该拥有完成任务所需的访问权。

例子：

- 日常工作用普通用户，不用管理员。
- 服务账号权限受限。
- 文件共享只给正确组。
- 定期检查应用权限。

初学者版本：

不要因为方便就直接给 “full control”。

## 基础加固清单

- [ ] 安装 OS 更新。
- [ ] 移除不需要的软件。
- [ ] 禁用不需要的服务。
- [ ] 检查本地管理员用户。
- [ ] 开启主机防火墙。
- [ ] 开启内置安全工具。
- [ ] 在合适场景开启磁盘加密。

## 按操作系统加固

### Windows

需要检查：

- Windows Update
- Microsoft Defender
- Windows Firewall
- Local Administrators group
- Remote Desktop 状态
- BitLocker
- Event logs
- 启动程序和计划任务

### Linux

需要检查：

- 软件包更新
- SSH 配置
- `sudo` 权限
- 监听中的服务
- 防火墙规则
- `/var/log`
- 文件权限
- 不用的用户

### macOS

需要检查：

- 系统更新
- FileVault
- Gatekeeper
- 防火墙
- 隐私权限
- Keychain 提示
- LaunchAgents 和 LaunchDaemons

## 我容易忘的点

- 加固不是一个设置。
- 方便通常会增加风险。
- 备份需要恢复测试。
- 如果什么都允许，防火墙帮助不大。
- 管理员访问应该少见，而不是日常默认。
- 打补丁很无聊，直到旧漏洞被利用。
