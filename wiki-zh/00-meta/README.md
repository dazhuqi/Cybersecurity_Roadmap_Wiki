# 00 元信息

| 字段 | 值 |
| --- | --- |
| 状态 | 使用中 |
| 用途 | 学习系统、进度追踪、复盘和 wiki 维护 |
| 最后复习 | 2026-07-01 |

这个目录管理学习过程，而不是某一个具体技术主题。技术笔记解释网络安全概念，meta 笔记帮助我决定学什么、复习什么，以及哪里还是弱点。

新手版理解：

> 这个文件夹是 wiki 的控制面板。

## 页面索引

| 页面 | 用途 | 什么时候更新 |
| --- | --- | --- |
| [progress-tracker.md](progress-tracker.md) | 总体路线图状态和下一步 | 至少每周 |

## 我应该怎么用这个文件夹

- 每周从 [progress-tracker.md](progress-tracker.md) 开始。
- 每次学习后使用 [learning-log.md](learning-log.md)。
- 把反复出现的术语放进 [glossary.md](glossary.md)。
- 每周结束写 [weekly-review.md](weekly-review.md)。
- 新增文件夹或规则时更新 [wiki-structure.md](wiki-structure.md)。

## 每周维护流程

1. 检查 progress tracker。
2. 选择一个当前模块重点。
3. 查看未完成笔记。
4. 把不清楚的术语移动到 glossary。
5. 至少把一个实验或阅读 session 变成具体笔记。
6. 写 weekly review。

## 好的追踪是什么样

好的追踪应该短，但诚实：

- 我学了什么
- 我实际练了什么
- 什么还不清楚
- 下一步做什么

坏追踪只是列出打开过的资源。打开页面不等于学会。

## 给未来自己的规则

- 如果不能不看笔记解释主题，不要标记完成。
- 不要隐藏困惑。困惑是有用数据。
- 当一个练习任务更有用时，不要再收集五个新资源。
- 主题重叠时互相链接。
- 模板要实用、容易填写。

## 维护边界

这个文件夹应该保持一种“有点无聊但很好用”的状态。它是为了让学习更轻松，不是为了变成另一个要折腾的项目。

如果我花在整理 tracker 上的时间比学习还多，就应该停下来选一个具体任务：

- 重写一篇看不懂的笔记
- 完成一个小 lab
- 复习一个 glossary 小节
- 关闭一个没用上的资源
- 把一个概念大声讲一遍

这个文件夹最好的样子不是花哨，而是我真的会更新。
