## 笔记

OSCP 是 OffSec 和 PEN-200 课程关联的实战渗透测试认证。它出名是因为更强调 hands-on exploitation 和报告，而不是只考选择题知识。

对我来说，OSCP 应该是长期实操目标，不应该急着冲。我需要先打好 Linux、Windows、网络、Web 基础、枚举习惯和报告写作。

## 为什么它和我的路线图有关

OSCP 风格准备连接很多路线图部分：

- Linux 和 Windows 基础
- 网络和排错
- Web 安全基础
- exploit research 和安全验证
- 权限提升
- Active Directory 基础
- 文档和报告

最大的教训是方法论。我要学会在路径不明显时继续推进。

## 我应该学到什么

- 仔细枚举。
- 发现和验证漏洞。
- 在授权实验环境里利用漏洞。
- Linux 和 Windows 权限提升。
- pivoting 和网络意识。
- 基础 Active Directory 攻击路径。
- 带证据和修复建议的清晰报告。

## 我会怎么准备

1. 完成 OS、网络、编程基础和 Web 安全基础。
2. 给每台实验机器写报告，养成习惯。
3. 在授权平台完成初级和中级实验。
4. 分别练 Linux 和 Windows 权限提升。
5. 计划考试前阅读官方考试指南。

## 练习想法

- 建个人枚举清单。
- 先不看 walkthrough 解 easy boxes，之后再对比笔记。
- 每个实验都写“我漏掉了什么”部分。
- 练习把 proof steps 写到别人能复现。
- 给 rabbit hole 限时，并记录为什么放弃。

## 初学者容易混的点

- OSCP-style 练习奖励耐心。快速乱猜通常比认真 enumeration 更浪费时间。
- 我应该边做边记笔记，而不是事后忘了关键细节再补。
- 拿到 shell 不是结束，还需要 proof、privilege escalation、cleanup awareness 和 reporting。
- privilege escalation 很依赖操作系统基础，Linux 或 Windows 太弱会拖慢整条路线。
- 挫败感是这条路线的一部分，所以我需要有退一步整理思路的流程，而不是随机敲命令。

## 小检查点

- 我能 enumerate 一个 lab target，并解释每一步为什么要做。
- 我能保留之后还能看懂的 command log。
- 我能先不看 walkthrough，独立解决 beginner boxes。
- 我能为一台 practice machine 写完整报告。
- 我能复盘一次失败尝试，并找出漏掉的线索。
