## 为什么这个主题重要

网络安全最后还是跑在真实的计算机上。哪怕后面学的是日志、恶意软件、云账号、网络流量，底下也一定有硬件在工作。如果我连电脑的基本组成部分都不理解，就很容易把问题发生的位置想错。

比如“电脑很慢”这句话可能有很多原因：内存太少、硬盘快坏了、开机启动项太多、恶意软件、过热降频，或者其实是网络卡。它们看起来都像慢，但解决方式完全不一样。硬件知识不会自动解决问题，但能让我一开始问更好的问题。

## 计算机硬件组件

### CPU

CPU，也就是中央处理器，负责执行指令。我可以先把它想成电脑里的主要“干活的人”：程序给它指令，它一步一步执行，只是速度非常快。

初学者要记住的点：

- CPU 核心数多通常有利于多任务，但不是每个程序都会很好地利用很多核心。
- CPU 过热时可能会自动降频，电脑就会变慢。
- 一些安全工具、虚拟机、扫描任务会吃很多 CPU。
- 如果 CPU 长时间接近 100%，系统可能会像卡死一样，虽然硬件不一定真的坏了。

安全角度：

- 恶意软件可能大量消耗 CPU，比如挖矿、加密勒索、解包、扫描文件。
- 某个可疑进程 CPU 很高不等于一定是恶意软件，但值得继续查。
- 虚拟机要从宿主机分配 CPU，CPU 太弱的话实验环境会很难受。

### RAM

RAM 是临时内存，程序运行时会用到它。电脑关机后，普通 RAM 里的内容就没了。

我经常把 RAM 和存储混在一起，所以这个比喻比较好记：

- RAM 像桌面空间。
- 存储像文件柜。
- 桌面越大，同时摊开的东西越多。

如果 RAM 不够，电脑可能会把硬盘当成临时内存用，这通常慢很多，尤其是旧机械硬盘。

安全角度：

- 系统运行时，内存里可能有敏感数据。
- 事件响应人员有时会采集内存镜像，用来查进程、网络连接、恶意代码和一些残留信息。
- 有些恶意软件尽量只在内存里运行，减少在硬盘上留下明显文件。

### 存储

存储负责在电脑关机后继续保存数据。常见类型是 HDD 和 SSD。

HDD 有机械结构，通常更慢；SSD 没有旋转磁盘，通常快很多。日常使用里，把旧电脑从 HDD 换成 SSD，体感可能像换了一台电脑。

我应该知道：

- 操作系统装在存储里。
- 用户文件保存在存储里。
- 日志也保存在存储里。
- 删除的文件不一定立刻彻底消失，是否能恢复取决于删除方式和是否被覆盖。
- 硬盘健康很重要，硬盘故障可能导致随机崩溃、启动很慢、文件损坏。

安全角度：

- 全盘加密能在设备丢失时保护数据。
- 备份能对抗硬件故障和勒索软件。
- 取证经常会先制作磁盘镜像，这样调查时不会直接改动原始磁盘。

### 主板

主板把主要组件连接在一起。CPU、RAM、存储、电源、扩展卡、各种接口都通过主板进行连接。

我不需要一开始就懂每条电路，但要知道主板提供了组件之间通信的路径。

安全角度：

- UEFI/BIOS 里的启动顺序很重要。
- Secure Boot 和 TPM 都跟启动时的信任有关。
- 如果电脑能随便从外部 USB 启动，这对维修有用，但如果物理访问没有控制，也可能有风险。

### 网卡

网卡让电脑能和网络通信。它可以是有线 Ethernet，也可以是 WiFi，或者两者都有。

排查时常看这些问题：

- 这个接口有没有 IP 地址？
- 它连的是不是预期网络？
- 当前用的是有线还是无线？
- 有没有默认网关？
- DNS 配置对不对？

安全角度：

- 网卡有 MAC 地址。
- 网络监控经常从接口流量开始。
- 如果设备有多个网络接口，流量可能会走我没想到的路径。

### 电源和电池

台式机靠电源供应器给组件供电，笔记本还会涉及电池和充电器。

电源问题有时候表现很奇怪。充电器功率不够、电源老化，都可能导致随机关机、无法启动、硬件不稳定。

安全角度：

- 可用性也是安全的一部分。系统如果无法稳定供电，就不可靠。
- 服务器和网络设备通常需要 UPS。
- 突然断电可能损坏数据，也可能中断日志记录。

## 常见连接类型

连接类型就是设备之间或设备到网络之间怎么通信。我不能只背名字，要知道它们的距离、速度、有线还是无线、常见用途。

### Ethernet

Ethernet 是有线网络连接，家庭和办公室里通常是 RJ45 网线。

为什么重要：

- 通常比 WiFi 稳定。
- 台式机、服务器、路由器、交换机、AP 都经常用它。
- 有线连接更容易理解，因为线缆能看到物理路径。

安全角度：

- 如果没有控制，插进一个 Ethernet 口可能就能进入内部网络。
- NAC 可以限制未知设备能做什么。
- 做安全检查时不能忽略物理端口。

### WiFi

WiFi 是用无线电信号连接网络。它方便，因为设备不用插线也能移动。

初学者笔记：

- WiFi 常见频段有 2.4 GHz、5 GHz，有些新设备还有 6 GHz。
- 2.4 GHz 覆盖更远，但更拥挤。
- 5 GHz 通常更快，但穿墙能力可能差一些。
- 路由器或无线 AP 会创建无线网络。

安全角度：

- 弱 WiFi 密码是真风险。
- WEP 这种旧加密不安全。
- Rogue AP 和 Evil Twin 攻击会利用用户对网络名称的信任。
- 访客网络可以把访客设备和内部设备隔离开。

### Bluetooth

Bluetooth 是短距离无线通信，常见于耳机、键盘、鼠标、手表、手机、车载系统。

它不是 WiFi。WiFi 通常用于网络和互联网访问，Bluetooth 更多是附近设备之间的直接连接。

安全角度：

- 一直保持可发现状态会增加暴露面。
- 在公共场所配对设备要小心。
- 老设备或没更新固件的设备可能存在蓝牙漏洞。
- 不用蓝牙时关掉，能减少不必要的攻击面。

### NFC

NFC 是近场通信，距离通常只有几厘米。非接触支付、门禁卡这类场景经常用到类似技术。

距离短是优点，因为设备需要非常靠近。但“距离短”不等于“完全没有风险”。

安全角度：

- NFC 常和支付、门禁、身份有关。
- 手机或卡片丢失后，如果仍能授权操作，就会有风险。
- 我需要理解便利和信任不是同一件事。

### Infrared

Infrared 是红外通信，用光进行短距离传输。电视遥控器是最容易理解的例子。

它通常需要视线无遮挡。中间有东西挡住，可能就不能用了。

安全角度：

- 红外在现代计算机网络里不是核心，但仍会出现在遥控器和一些设备通信里。
- 它提醒我：无线不一定都是无线电。

### Fiber

Fiber 是光纤连接，通过光在光纤里传输数据，常见于高速、长距离网络。

安全角度：

- 光纤常用于骨干网络和企业网络。
- 网络线缆的物理访问仍然重要。
- 光纤故障排查可能需要和铜缆 Ethernet 不一样的工具。

### Cellular

Cellular 是蜂窝移动网络，比如 4G、5G。手机、平板、热点、一部分笔记本可以使用它。

安全角度：

- 设备如果用自己的蜂窝网络，可能绕过公司 WiFi 监控。
- 工作数据在手机上时，移动设备管理就很重要。
- 手机丢失可能变成数据泄露事件。

## 我会怎么排查连接问题

如果设备连不上，我应该先慢下来问：

1. 是物理连接问题吗？
2. 网卡有没有启用？
3. 设备有没有 IP 地址？
4. 有没有默认网关？
5. DNS 能不能工作？
6. 是只有一个网站坏了，还是整个互联网都不通？
7. 是本机问题、路由器问题，还是上游网络问题？

不要一次改五个设置。因为就算修好了，也不知道到底是哪一步修好的。

## 小练习

- [ ] 打开系统信息，记录 CPU、RAM、存储类型。

- [ ] 看看路由器或 AP，分清 WAN 口和 LAN 口。

## 我容易忘的点

- RAM 是临时的，存储是持久的。
- WiFi 和 Bluetooth 都是无线，但解决的问题不一样。
- 插了网线不代表网络配置一定正确。
- 物理访问也是安全问题，不只是 IT 支持问题。
- 电脑慢不等于一定中毒。
