# 书籍

书比视频慢一点，但它的好处是解释更完整。我的目标不是一次把这些书全读完，而是选一本和当前模块有关的书，边读边做笔记，再把知识放进实验里验证。

## 我怎么选一本书

开始读之前，我应该先写清楚：

- 这本书对应路线图里的哪个模块。
- 我想通过它解决什么问题。
- 我到底要读完整本，还是只读几个章节。
- 读完以后准备用什么实验或命令练一下。

如果这些问题答不上来，那我可能只是在收集书单，不是在学习。

## 基础能力

| 书名 | 主题 | 难度 | 链接 | 备注 |
| --- | --- | --- | --- | --- |
| How Linux Works, 3rd Edition | Linux | 入门到中级 | [No Starch Press](https://nostarch.com/howlinuxworks3) | 适合理解 Linux，而不只是复制命令。和操作系统模块很搭。 |
| Linux Basics for Hackers | Linux 和安全工具 | 入门 | [No Starch Press](https://nostarch.com/linuxbasicsforhackers) | 比较友好，但我还是要验证命令，不要把工具当魔法。 |
| Computer Networking: A Top-Down Approach | 网络 | 入门到中级 | [Book website](https://gaia.cs.umass.edu/kurose_ross/) | 适合理解互联网到底怎么工作。比快速考证指南更偏教材。 |
| Python Crash Course, 3rd Edition | Python 编程 | 入门 | [No Starch Press](https://nostarch.com/python-crash-course-3rd-edition) | 如果我要先补 Python 基础和小项目，这本比较适合。 |
| Automate the Boring Stuff with Python | Python 自动化 | 入门 | [Book website](https://automatetheboringstuff.com/) | 适合练文件处理、正则、自动化脚本这些实用内容。 |

## 安全基础

| 书名 | 主题 | 难度 | 链接 | 备注 |
| --- | --- | --- | --- | --- |
| Security Engineering, 3rd Edition | 安全工程 | 中级 | [Author page](https://www.cl.cam.ac.uk/archive/rja14/book.html) | 内容很大很广，更像长期参考书，不适合一开始硬啃。 |

## 我应该怎么读

- 不要被动阅读。每章或每节后写一小段笔记。
- 读完后尝试一个相关命令、实验或图。
- 把困惑点标出来，不要假装已经懂了。
- 当对应模块变成当前重点时，再回来读相关章节。
- 如果后面章节太难，可以先停在有用的部分，不要为了完成而硬读。
