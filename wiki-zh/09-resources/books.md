# 书籍

书比视频慢，但当我需要更完整解释时很有用。我不应该试图一次读完所有这些书。选一本和当前模块匹配的书，做笔记，再把内容连接到实验里。

## 阅读清单

| 书名 | 主题 | 难度 | 链接 | 备注 |
| --- | --- | --- | --- | --- |
| How Linux Works, 3rd Edition | Linux | 入门到中级 | [No Starch Press](https://nostarch.com/howlinuxworks3) | 适合理解 Linux，不只是复制命令。和操作系统模块很配。 |
| Linux Basics for Hackers | Linux 和安全工具 | 入门 | [No Starch Press](https://nostarch.com/linuxbasicsforhackers) | 比较友好，但我还是要验证命令，不要把它当魔法。 |
| Computer Networking: A Top-Down Approach | 网络 | 入门到中级 | [Book website](https://gaia.cs.umass.edu/kurose_ross/) | 适合理解互联网怎么工作。比快速考证指南更偏学术。 |
| Serious Cryptography | 密码学 | 中级 | [No Starch Press](https://nostarch.com/seriouscrypto) | 基础安全概念之后再读，适合理解为什么 crypto 错误很严重。 |
| Real-World Cryptography | 密码学 | 中级 | [Manning](https://www.manning.com/books/real-world-cryptography) | 更实践一点的密码学解释，不是第一周就读的书。 |
| Practical Malware Analysis | 恶意软件分析 | 高级 | [No Starch Press](https://nostarch.com/malware) | 经典书，但要等 Windows、汇编基础和安全实验室习惯更强后再看。 |
| Web Security Testing Guide | Web 安全测试 | 入门到中级 | [OWASP WSTG](https://owasp.org/www-project-web-security-testing-guide/) | 更像长篇指南，不是普通书。学 Web 测试方法论时很有用。 |

## 我应该怎么读

- 不要被动阅读。每章或每节后写一小段笔记。
- 读完后尝试一个相关命令、实验或图。
- 把困惑点标出来，不要假装已经懂了。
- 当对应模块变成当前重点时再回来看相关书。

## 按当前阶段优先级

| 阶段 | 更适合的选择 |
| --- | --- |
| 基础 IT 和 OS | How Linux Works |
| 网络基础 | Computer Networking: A Top-Down Approach |
| Web 安全 | OWASP Web Security Testing Guide |
| 密码学基础 | Serious Cryptography 或 Real-World Cryptography |
| 后期恶意软件和逆向 | Practical Malware Analysis |
