# 09 资源

| 字段 | 值 |
| --- | --- |
| 状态 | 参考 |
| 路线图区域 | Resources |
| 最后复习 | 2026-07-01 |

这个目录收集支持路线图的资源。它不应该变成一堆永远不会用的链接。每个资源都应该有目的：学习概念、练习技能、查命令、比较工具，或者找官方指导。

新手版理解：

> 资源只有在帮助当前学习目标时才有用。

## 资源页面

| 页面 | 用来做什么 | 规则 |
| --- | --- | --- |
| [books.md](books.md) | 更慢、更深入的解释 | 当前模块选一本书，不要一次开十本。 |
| [courses.md](courses.md) | 结构化学习路径 | 每个小节后要练习。 |
| [tools.md](tools.md) | 工具参考和实验工具 | 使用工具前先知道自己在问什么问题。 |
| [websites.md](websites.md) | 官方文档和稳定参考 | 事实和配置优先看官方来源。 |
| [cheatsheets.md](cheatsheets.md) | 快速语法提醒 | 不理解就不要复制命令。 |

## 怎么选择资源

打开资源前先问：

- 它支持路线图哪个模块？
- 我想回答什么问题？
- 读完后会不会练习？
- 它是官方文档、课程、博客，还是个人笔记？
- 对这个主题来说，它是否足够新？

如果答不上来，我可能是在用收藏链接逃避练习。

## 资源优先级

| 场景 | 最好从哪里开始 |
| --- | --- |
| 需要准确产品行为 | 官方文档 |
| 需要新手解释 | 课程、learning center 或书 |
| 忘记命令语法 | cheatsheet 或 manual page |
| 需要练习 | lab platform 或 exercise |
| 需要安全方法论 | OWASP、NIST、CISA、MITRE、vendor guidance |
| 需要当前漏洞事实 | CVE/NVD、vendor advisory、CISA KEV、official source |

## 给未来自己的提醒

- 不要添加没有用途说明的资源。
- 不要因为课程流行就开始付费课程。
- 不要把工具当技能。
- 不要不看日期就相信旧安全文章。
- 把真正有用的资源链接到使用它的模块页面。

## 复习问题

- 这周我实际用了哪个资源？
- 它有没有带来练习，还是只是更多阅读？
- 我是否保存了具体页面链接？
- 是否有更好的官方来源？
- 这个资源应该删除还是先停放？
