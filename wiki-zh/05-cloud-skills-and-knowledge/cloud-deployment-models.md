# 云部署模型

| 字段 | 值 |
| --- | --- |
| 状态 | 学习中 |
| 路线图区域 | Cloud Skills and Knowledge |
| 最后复习 | 2026-06-30 |

云部署模型描述云环境运行在哪里、给谁使用。它会影响控制权、成本、信任边界、网络设计、合规和运维。

新手版理解：

> 服务模型问的是“这是什么类型的服务？”部署模型问的是“它部署在哪里、给谁用？”

## 主要部署模型

路线图重点是：

- 公有云
- 私有云
- 混合云

NIST 还提到 community cloud。它在新手例子里没那么常见，但知道这个概念有帮助。

## 公有云

公有云表示云服务商通过共享的服务商基础设施，为很多客户提供服务。

例子：

- AWS
- Microsoft Azure
- Google Cloud
- Oracle Cloud Infrastructure

重要点：

“共享基础设施”不代表每个客户都能看到别人的数据。服务商会使用隔离控制、身份边界、网络分段和虚拟化。但我仍然要安全配置自己的账号。

常见用途：

- 托管网站
- 运行虚拟机
- 存储备份
- 构建数据流水线
- 测试和开发实验室
- serverless 应用

安全优点：

- 服务商管理物理安全
- 服务商提供很多内置安全服务
- 容易创建隔离环境
- 有日志和监控工具
- 扩展和备份选项更容易使用

安全风险：

- 很容易快速暴露到互联网
- IAM 错误影响很大
- 公开存储错误很常见
- 服务创建速度可能超过治理速度
- 成本和资产清单可能变乱

## 私有云

私有云表示只给一个组织使用的云风格基础设施。它可以运行在组织自己的数据中心，也可以由第三方托管。

例子：

- 基于虚拟化搭建的内部云平台
- 只给一个组织使用的 OpenStack
- VMware-based private cloud
- hosted private cloud

常见用途：

- 严格监管或数据位置要求
- 旧应用不容易迁移
- 组织已经有数据中心
- 工作负载需要自定义基础设施控制

安全优点：

- 对基础设施有更直接的控制
- 更容易贴合某些内部政策
- 多租户顾虑更少
- 可以和现有本地控制集成

安全风险：

- 组织要管理更多基础设施
- patching 和平台维护可能更难
- 托管安全服务可能不如公有云多
- private 不自动等于 secure
- 旧的本地机房习惯可能被带过来

## 混合云

混合云把两个或更多环境连接起来，比如本地系统加公有云，或者私有云加公有云。

例子：

- 本地 Active Directory 连接到 Azure 服务
- 数据库留在本地，Web app 放在 AWS
- 灾备环境放在公有云
- 云分析服务读取本地系统数据

常见用途：

- 分阶段迁移
- 灾难恢复
- 敏感工作负载留在本地
- 用云服务做突发容量
- 让旧系统连接现代云应用

安全优点：

- 迁移路径灵活
- 可以把部分数据或系统留在更受控的位置
- 不需要一次性迁移所有东西也能使用云安全工具

安全风险：

- 网络路径更复杂
- identity federation 容易配置错
- 团队之间责任不清
- 日志可能分散在多个环境
- 如果信任太宽，攻击者可能在本地和云之间移动

## Community Cloud

Community cloud 是由几个有共同需求的组织共享的云，比如政府、医疗、研究或教育。

新手提醒：

我现在不需要优先掌握它，但它能提醒我：云不只是“公有 vs 私有”。有些环境是给某类共同需求的组织共享的。

## 对比表

| 模型 | 谁使用 | 主要优点 | 主要安全挑战 |
| --- | --- | --- | --- |
| 公有云 | 很多客户 | 速度、规模、托管服务 | 配置错误、IAM、公开暴露 |
| 私有云 | 一个组织 | 控制和自定义 | 维护、补丁、内部运维 |
| 混合云 | 连接的多个环境 | 灵活迁移 | 复杂性、身份、网络信任 |
| Community cloud | 类似需求的组织 | 共同要求 | 治理和共享边界 |

## Public vs Private 不等于 Secure vs Insecure

这是新手陷阱。

公有云配置得好可以很安全。私有云如果没打补丁、监控差、账号权限过大，也可能很不安全。

更好的问题是：

- 谁能访问这个环境？
- 身份怎么控制？
- 什么暴露在互联网上？
- 日志存在哪里？
- 谁负责 patch 平台？
- secrets 怎么管理？
- 谁关注配置漂移？

## 混合云安全边界

混合云需要特别小心，因为它连接多个环境。

需要检查：

- VPN 或私有连接
- 网络之间的路由
- 跨环境 DNS 解析
- identity federation
- 特权账号
- 两边的日志
- 数据传输规则
- firewall 和 security group rules

新手版理解：

混合云会搭桥。桥很有用，但错误和攻击者也可能从桥上过去。

## 我总是忘的点

- 部署模型说的是“在哪里、给谁”，不是 SaaS/PaaS/IaaS。
- 公有云是多租户，但服务商会做隔离控制。
- 私有云给了控制，也给了更多责任。
- 混合云很强，但很容易变复杂。
- 混合架构里，连接和身份是风险重点。
