# 03 Networking Knowledge

Networking is the part of cybersecurity that makes many other topics less mysterious. When I understand networking better, logs, scans, firewalls, VPNs, DNS, web traffic, and packet captures start to feel connected instead of random.

This module is not trying to make me a network engineer immediately. It is trying to build enough beginner understanding so that I can look at an IP address, port, protocol, or packet capture and not panic.

## Learning Order

1. [osi-and-tcp-ip.md](osi-and-tcp-ip.md)
2. [ip-addressing-subnetting.md](ip-addressing-subnetting.md)
3. [network-services.md](network-services.md)
4. [protocols-and-ports.md](protocols-and-ports.md)
5. [network-devices-topologies.md](network-devices-topologies.md)
6. [wireless-and-vpn.md](wireless-and-vpn.md)
7. [troubleshooting-tools.md](troubleshooting-tools.md)

I put the OSI/TCP-IP models first because they give me a map. IP addressing comes next because every network conversation needs addressing. Services, protocols, and ports come after that. Devices, wireless, tools, and packet analysis make more sense once the basics are in place.

## What I Should Be Able to Explain

After finishing this module, I should be able to explain:

- Why network models are useful.
- What an IP address is.
- What a subnet is at a beginner level.
- What DNS, DHCP, and NTP do.
- Why protocols use ports.
- What routers, switches, firewalls, VLANs, and DMZs are for.
- How WiFi and VPNs fit into security.
- How to use basic troubleshooting commands.
- What a packet capture can show.

## My Study Method

For each topic, I will try to write:

- A plain-English explanation.
- A small example.
- A security angle.
- Commands or tools if relevant.
- Review questions.

Networking has many words that sound similar. I will not pretend they are obvious. If two terms are easy to confuse, I will write the difference directly.

## Completion Checklist

- [ ] I can explain OSI and TCP/IP without memorizing every detail perfectly.
- [ ] I can identify private IP addresses.
- [ ] I can explain CIDR notation at a beginner level.
- [ ] I can explain DNS and DHCP.
- [ ] I can connect common ports to services.
- [ ] I can explain the difference between a router and a switch.
- [ ] I can run basic network troubleshooting commands.
- [ ] I can open a packet capture and find DNS or HTTP traffic.
