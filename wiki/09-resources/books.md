# Books

Books are slower than videos, but they are good when I need a fuller explanation. I should not try to read all of these at once. Pick one book that matches the current module, take notes, and connect it to labs.

## Reading List

| Title | Topic | Level | Link | Notes |
| --- | --- | --- | --- | --- |
| How Linux Works, 3rd Edition | Linux | Beginner to intermediate | [No Starch Press](https://nostarch.com/howlinuxworks3) | Good for understanding Linux beyond copying commands. Useful with the OS module. |
| Linux Basics for Hackers | Linux and security tools | Beginner | [No Starch Press](https://nostarch.com/linuxbasicsforhackers) | Friendly entry point, but I should still verify commands and not treat it as magic. |
| Computer Networking: A Top-Down Approach | Networking | Beginner to intermediate | [Book website](https://gaia.cs.umass.edu/kurose_ross/) | Strong for understanding how the internet actually works. More academic than a quick cert guide. |
| Serious Cryptography | Cryptography | Intermediate | [No Starch Press](https://nostarch.com/seriouscrypto) | Read after basic security concepts. Good for learning why crypto mistakes matter. |
| Real-World Cryptography | Cryptography | Intermediate | [Manning](https://www.manning.com/books/real-world-cryptography) | Practical crypto explanations. Not a first-week book. |
| Practical Malware Analysis | Malware analysis | Advanced | [No Starch Press](https://nostarch.com/malware) | Classic book, but I should wait until Windows, assembly basics, and safe lab habits are stronger. |
| Web Security Testing Guide | Web security testing | Beginner to intermediate | [OWASP WSTG](https://owasp.org/www-project-web-security-testing-guide/) | More like a long guide than a normal book. Useful when learning web testing methodology. |

## How I Should Read

- Do not read passively. Make a small note after each chapter or section.
- Try one related command, lab, or diagram after reading.
- Mark confusing parts instead of pretending they are clear.
- Revisit books when the matching module becomes active.
