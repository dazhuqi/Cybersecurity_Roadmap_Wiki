# Courses

Courses are useful when I need structure. The risk is watching too much and practicing too little. For each course, I should write down what I actually did after watching or reading.

## Course List

| Course | Provider | Topic | Level | Status | Notes |
| --- | --- | --- | --- | --- | --- |
| Cyber Security Roadmap | roadmap.sh | Overall roadmap | Beginner | Reference | Use this as the structure, not as a checklist to rush. |
| Pre Security / Beginner Paths | TryHackMe | IT, networking, web, security basics | Beginner | Not Started | Good when I need guided labs instead of only reading. |
| Hack The Box Academy | Hack The Box | Security modules and practical skills | Beginner to advanced | Not Started | Use modules when I want structured explanation before machines. |
| Web Security Academy | PortSwigger | Web application security | Beginner to advanced | Not Started | Best web security practice source for request/response thinking. |
| Networking Basics | Cisco Networking Academy | Networking | Beginner | Not Started | Useful before deeper CCNA-style study. |
| Introduction to Linux | Linux Foundation | Linux basics | Beginner | Not Started | Good supplement for the Linux module. |
| Microsoft Learn Security | Microsoft Learn | Microsoft security and cloud security | Beginner to intermediate | Reference | Use official docs when learning Windows, Defender, Entra, or Azure security. |
| AWS Skill Builder Security | AWS Skill Builder | Cloud security | Beginner to intermediate | Later | Save for the cloud module. |
| OpenSecurityTraining2 | OST2 | Low-level security and reverse engineering | Intermediate to advanced | Later | Excellent but not a beginner starting point. |

## Rule for Myself
