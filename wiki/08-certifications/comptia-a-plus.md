## Notes

CompTIA A+ is an entry-level IT certification. It is not a cybersecurity certification by itself, but it covers many things that security people still need to understand: hardware, operating systems, networking basics, mobile devices, troubleshooting, and support workflows.

I may or may not actually sit the exam. Even if I do not, A+ style knowledge is useful because it fills the "normal IT" gaps that can make security topics harder later.

## Why It Matters for My Roadmap

A lot of security work starts with basic IT observations:

- What device is this?
- What operating system is running?
- Is the problem hardware, software, network, account, or user workflow?
- What changed recently?
- What logs or settings should I check first?

If I skip these foundations, I may jump to "malware" or "attack" too quickly when the real issue is a normal IT problem.

## What I Should Learn From It

- PC hardware components and common failure symptoms.
- Windows, macOS, Linux, mobile, and basic OS installation ideas.
- Basic networking, WiFi, IP settings, and troubleshooting.
- Security basics such as authentication, malware awareness, and least privilege.
- Ticketing, documentation, and careful troubleshooting habits.

## How I Would Prepare

1. Read the official exam objectives from CompTIA.
2. Match weak topics to my notes in `01-fundamental-it-skills` and `02-operating-systems`.
3. Build a small checklist for hardware, OS, and network troubleshooting.
4. Do practice questions only after I understand the concepts.
5. Use a VM or spare machine for hands-on OS practice.

## Hands-On Practice Ideas

- Identify CPU, RAM, storage, network adapter, and OS version on my machine.
- Install a Linux VM and record the basic setup steps.
- Practice `ipconfig`, `ping`, `nslookup`, and Event Viewer on Windows.
- Practice basic file navigation and permissions on Linux.
- Write one troubleshooting note using the template from the IT skills module.

## Beginner Things I Need to Be Careful About

- Do not memorize hardware names without knowing what job the part does.
- BIOS/UEFI is firmware. Windows or Linux is the operating system. I should not mix those up.
- RAM is short-term working memory. Storage keeps data after power is off.
- Troubleshooting is not guessing faster. It is checking simple causes first, changing one thing at a time, and writing down what happened.
- Customer support skill matters here. A technically correct answer can still be bad if I explain it in a confusing or impatient way.

## Small Checkpoints

- I can describe the boot process in beginner language.
- I can explain what CPU, RAM, storage, motherboard, GPU, and PSU do.
- I can install or update a driver in a lab machine and record what changed.
- I can read a basic error message and search for the exact wording.
- I can make a simple backup and restore plan for a personal computer.
