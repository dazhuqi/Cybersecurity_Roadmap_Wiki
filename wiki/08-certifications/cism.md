## Notes

CISM stands for Certified Information Security Manager. It is focused on managing security programs, risk, governance, and incident management. It is not mainly about configuring tools or exploiting systems.

This is a later-career certification for me unless I move into security management early. Still, it is useful to understand because technical work eventually has to support business risk decisions.

## Why It Matters for My Roadmap

CISM-style knowledge helps explain questions like:

- What security program are we building?
- Which risks matter most?
- How do policies, people, and processes support technical controls?
- How should incidents be managed at an organizational level?
- How do we measure whether security work is effective?

This reminds me that security is not only "find vulnerability, fix vulnerability."

## What I Should Learn From It

- Information security governance.
- Information risk management.
- Security program development and management.
- Incident management and response coordination.
- Communication with leadership and stakeholders.
- Policy and process thinking.

## How I Would Prepare

1. Build technical foundations first so management ideas have context.
2. Learn risk vocabulary and how security decisions affect business operations.
3. Read official ISACA CISM information and requirements.
4. Practice writing short executive summaries for technical issues.
5. Compare CISM with CISA and CISSP before choosing a path.

## Practice Ideas

- Turn a technical finding into a management-level risk statement.
- Write a simple incident communication plan.
- Map one security control to business impact.
- Review the difference between governance, management, and operations.
