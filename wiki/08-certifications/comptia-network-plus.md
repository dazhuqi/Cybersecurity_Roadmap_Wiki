## Notes

CompTIA Network+ is a vendor-neutral networking certification. It is useful because networking shows up almost everywhere in cybersecurity: firewalls, scans, VPNs, DNS logs, packet captures, cloud networking, and incident response.

For me, Network+ is less about the certificate name and more about forcing myself to learn networking in a structured way.

## Why It Matters for My Roadmap

If I cannot explain IP addresses, ports, DNS, routing, and basic troubleshooting, many security topics will feel like random commands. Network+ style study can support:

- `03-networking-knowledge`
- basic SOC investigation
- firewall and VPN concepts
- vulnerability scanning
- web security basics
- packet analysis

## What I Should Learn From It

- OSI and TCP/IP models.
- IP addressing, subnet masks, CIDR, and private/public addressing.
- DNS, DHCP, NAT, NTP, VLANs, VPNs, and wireless basics.
- Common ports and protocols.
- Network devices such as switches, routers, firewalls, and access points.
- Troubleshooting tools like `ping`, `traceroute`, `nslookup`, `dig`, `ipconfig`, `ip`, `ss`, and `nmap`.

## How I Would Prepare

1. Finish the notes in `03-networking-knowledge`.
2. Draw my home or lab network and label devices, IPs, and gateways.
3. Practice subnetting until `/24`, `/25`, `/26`, and private ranges feel less scary.
4. Memorize common ports only after understanding the services.
5. Use Wireshark for basic DNS, TCP, HTTP, and TLS observation.

## Hands-On Practice Ideas

- Find my IP address, gateway, DNS server, and route table.
- Use `nslookup` or `dig` to compare DNS results.
- Use `traceroute` or `tracert` and write down what the output means.
- Capture one website visit in Wireshark and identify DNS plus TCP/TLS traffic.
- Build a small VM network and test NAT vs host-only networking.
