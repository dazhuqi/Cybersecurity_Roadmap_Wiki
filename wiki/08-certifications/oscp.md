## Notes

OSCP is OffSec's practical penetration testing certification tied to the PEN-200 course. It is widely known because it emphasizes hands-on exploitation and reporting instead of only multiple-choice knowledge.

For me, OSCP should be a long-term practical goal, not something to rush. I need strong Linux, Windows, networking, web basics, enumeration discipline, and report writing before taking it seriously.

## Why It Matters for My Roadmap

OSCP-style preparation connects many parts of the roadmap:

- Linux and Windows fundamentals
- networking and troubleshooting
- web security basics
- exploit research and safe validation
- privilege escalation
- Active Directory basics
- documentation and reporting

The biggest lesson is methodology. I need to learn how to keep moving when the path is not obvious.

## What I Should Learn From It

- Careful enumeration.
- Finding and validating vulnerabilities.
- Exploitation in authorized lab environments.
- Linux and Windows privilege escalation.
- Pivoting and network awareness.
- Basic Active Directory attack paths.
- Clean reporting with evidence and remediation.

## How I Would Prepare

1. Finish OS, networking, programming basics, and web security foundations.
2. Build a habit of writing reports for every lab machine.
3. Complete beginner and intermediate labs on authorized platforms.
4. Practice privilege escalation separately on Linux and Windows.
5. Read the official exam guide before planning any exam attempt.

## Practice Ideas

- Build a personal enumeration checklist.
- Solve easy boxes without walkthroughs first, then compare notes after.
- Keep a "things I missed" section for every lab.
- Practice writing proof steps clearly enough that another person could reproduce them.
- Time-box rabbit holes and record why I moved on.

## Beginner Things I Need to Be Careful About

- OSCP-style work rewards patience. Fast guessing usually wastes more time than careful enumeration.
- I should keep notes while working, not after I forget the important details.
- A shell is not the end. I still need proof, privilege escalation, cleanup awareness, and reporting.
- Privilege escalation depends on operating system basics. Weak Linux or Windows knowledge will slow everything down.
- Frustration is part of this path, so I need a process for stepping back instead of randomly trying commands.

## Small Checkpoints

- I can enumerate a lab target and explain why each step matters.
- I can keep a command log that is readable later.
- I can solve beginner boxes without watching walkthroughs first.
- I can write a full report for a practice machine.
- I can review a failed attempt and identify the missed clue.
