## Notes

GPEN stands for GIAC Penetration Tester. It focuses on penetration testing methodology, reconnaissance, exploitation concepts, password attacks, post-exploitation awareness, and reporting.

For me, GPEN is a serious offensive security direction, not a beginner shortcut. I should build strong networking, Linux, Windows, web, and lab habits first.

## Why It Matters for My Roadmap

GPEN-style study helps connect:

- authorized testing scope
- reconnaissance and enumeration
- vulnerability validation
- exploitation workflow
- privilege escalation awareness
- password attack concepts
- reporting and remediation

The key word is authorized. Penetration testing only makes sense inside clear permission and scope.

## What I Should Learn From It

- Penetration testing process and rules of engagement.
- Reconnaissance and scanning.
- Exploitation concepts and safe validation.
- Password attacks and credential handling.
- Windows and Linux post-exploitation basics.
- Reporting findings clearly.
- Ethics and legal boundaries.

## How I Would Prepare

1. Complete beginner labs on TryHackMe and Hack The Box.
2. Practice enumeration until it becomes methodical.
3. Write reports for solved lab machines.
4. Learn Windows and Linux privilege escalation basics in authorized labs.
5. Compare GPEN with OSCP and CREST before deciding.

## Practice Ideas

- Build an enumeration checklist.
- Run `nmap` against lab targets and explain each option used.
- Complete a beginner vulnerable VM and write a report.
- Practice turning technical proof into remediation advice.
- Keep a failed-attempts section in every lab note.

## Beginner Things I Need to Be Careful About

- GPEN is about penetration testing methodology, not only getting shells.
- A test has rules, scope, time limits, evidence, and reporting. Those parts matter.
- Enumeration usually decides whether exploitation works. Skipping it creates fake progress.
- Post-exploitation should stay inside scope and should be documented carefully.
- I should be able to explain risk and remediation, not only the command that worked.

## Small Checkpoints

- I can build a repeatable enumeration checklist for lab machines.
- I can explain what I tried, what failed, and what I learned.
- I can safely practice privilege escalation in intentionally vulnerable environments.
- I can write a short attack path from initial access to impact.
- I can produce a report that a defender could actually use.
