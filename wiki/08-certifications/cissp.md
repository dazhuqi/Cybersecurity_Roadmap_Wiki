## Notes

CISSP is a broad, senior-level security certification from ISC2. It covers security and risk management, asset security, architecture, engineering, communications, identity, assessment, operations, and software development security.

This is not a beginner target for me. It is more like a long-term map of what a well-rounded security professional is expected to understand.

## Why It Matters for My Roadmap

CISSP-style knowledge is useful because it connects technical work with bigger security architecture and management ideas:

- why controls exist
- how risk is handled
- how identity and access fit into architecture
- why operations, software, networks, and governance are connected
- how to think beyond one tool or one vulnerability

Even if I do not take it soon, its domains can help me see gaps.

## What I Should Learn From It

- Security and risk management.
- Asset security and data classification.
- Security architecture and engineering.
- Communication and network security.
- Identity and access management.
- Security assessment and testing.
- Security operations.
- Software development security.

## How I Would Prepare Later

1. Build broad hands-on experience first.
2. Use the official exam outline as a gap checklist.
3. Connect each domain to real examples from labs, work, or case studies.
4. Avoid studying only from flashcards because the topics need judgment.
5. Check ISC2 experience and endorsement requirements before planning.

## Practice Ideas

- Map a lab finding to CISSP-style domains.
- Write a short risk statement for a technical weakness.
- Review identity, access control, and data classification examples.
- Compare security architecture choices, such as segmentation and least privilege.
