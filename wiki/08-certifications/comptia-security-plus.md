## Notes

CompTIA Security+ is usually treated as a first broad cybersecurity certification. It covers many topics at a high level: threats, vulnerabilities, architecture, operations, risk, identity, cryptography, and incident response.

The danger is that I could memorize definitions without being able to use them. I want to treat Security+ as a checklist for security vocabulary, then connect each topic to a real example or lab.

## Why It Matters for My Roadmap

Security+ style study connects to:

- `04-security-skills-and-knowledge`
- basic blue team concepts
- incident response vocabulary
- risk and governance basics
- authentication and access control
- cryptography fundamentals
- vulnerability management

It is broad, so it can help me see the map before choosing a more specific path.

## What I Should Learn From It

- Common threat actors, attack types, and social engineering.
- Vulnerability scanning, patching, and secure configuration basics.
- Network, cloud, application, and endpoint security ideas.
- Identity and access management.
- Cryptography and PKI basics.
- Logging, monitoring, incident response, and business continuity.
- Risk, compliance, and security policies.

## How I Would Prepare

1. Read the official exam objectives.
2. Build a glossary for terms I cannot explain simply.
3. Match each concept to at least one example, command, log, or scenario.
4. Review networking and operating system basics before practice exams.
5. Use practice questions to find weak areas, not as the main learning method.

## Hands-On Practice Ideas

- Review Windows Defender, firewall, and Event Viewer on a test system.
- Run a basic vulnerability scan in an authorized lab.
- Write a simple incident response checklist.
- Compare MFA, password manager, and least privilege controls.
- Map common attacks to the security control that reduces the risk.
