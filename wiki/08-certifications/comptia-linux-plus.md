## Notes

CompTIA Linux+ focuses on Linux administration fundamentals. For cybersecurity, Linux matters because many servers, tools, containers, cloud systems, and lab machines use it.

This certification is not my first priority unless I need a Linux-focused credential, but the knowledge is very useful. I should be comfortable in Linux before serious security labs.

## Why It Matters for My Roadmap

Linux+ style study supports:

- `02-operating-systems/linux.md`
- command-line confidence
- server and cloud basics
- shell scripting foundations
- security tool usage
- log reading and service troubleshooting

If I am afraid of the terminal, many security labs will feel harder than they need to.

## What I Should Learn From It

- Linux file system layout and navigation.
- Users, groups, permissions, ownership, and `sudo`.
- Package management and software updates.
- Processes, services, systemd, and logs.
- Networking commands and SSH.
- Storage basics, shell scripting, and troubleshooting.
- Basic security hardening.

## How I Would Prepare

1. Use a Linux VM as a daily practice system for a while.
2. Finish the Linux, command-line, permissions, and hardening notes.
3. Build a personal command notebook with examples and outputs.
4. Practice breaking and fixing services in a lab.
5. Read logs instead of only running commands.

## Hands-On Practice Ideas

- Create users and groups in a Linux VM.
- Change file permissions and explain the result.
- Install, start, stop, enable, and disable a service.
- Read authentication logs and system logs.
- Configure SSH safely in a lab.
- Write a small Bash script that checks system information.
