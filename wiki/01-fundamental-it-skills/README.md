# 01 Fundamental IT Skills

This module is the "before cybersecurity" part of the roadmap. I used to think cybersecurity starts with hacking tools, but this section is a reminder that a lot of security work depends on normal IT knowledge: knowing what a computer is made of, how devices connect, how to troubleshoot boring problems, how office/cloud tools are used, and what a basic network looks like.

The notes in this folder are intentionally written from a beginner point of view. I am not trying to sound like an expert here. I want to make sure I can explain the basic ideas without skipping the obvious parts.

## Learning Order

1. [hardware-and-connectivity.md](hardware-and-connectivity.md)
2. [troubleshooting-and-productivity.md](troubleshooting-and-productivity.md)
3. [virtualization.md](virtualization.md)
4. [networking-basics.md](networking-basics.md)

I think this order makes sense because hardware and connections are the physical base, troubleshooting is the everyday working method, virtualization gives me a safe lab, and networking connects everything together.

## What I Should Be Able to Explain

After finishing this module, I should be able to explain:

- What CPU, RAM, storage, motherboard, network card, and power supply do.
- Why wired, wireless, short-range, and cloud connections are not the same thing.
- How to approach a problem before randomly changing settings.
- Why productivity suites are part of security learning.
- What a virtual machine is and why it is useful for labs.
- How a computer reaches a website at a high level.

## Study Method

For each note, I will try to write three kinds of information:

- Plain explanation: what the thing is in normal words.
- Security angle: why this matters for cybersecurity.
- Small practice: one small thing I can check or try on my own machine.

This is slower than only copying definitions, but it should make the notes easier to review later.

## Completion Checklist

- [ ] I can describe the main components inside a computer.
- [ ] I can name common connection types and what they are usually used for.
- [ ] I can follow a simple troubleshooting process.
