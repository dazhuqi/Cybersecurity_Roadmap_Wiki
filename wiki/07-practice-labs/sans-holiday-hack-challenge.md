SANS Holiday Hack Challenge is a seasonal challenge series. It is useful because it mixes technical tasks with a story, objectives, hints, and final reporting. That makes it feel closer to a small investigation than a single isolated puzzle.

I should use it for slower practice: take notes, collect evidence, explain each objective, and write a clean final report. This is a good place to practice communication, not only technical solving.

## Why It Is Useful

This challenge is helpful because it mixes:

- web tasks
- forensics
- cloud or identity ideas
- command-line puzzles
- investigation notes
- report writing
- hints and narrative context

The report-writing part matters. In real work, solving quietly is not enough. I need to explain what happened and why it matters.

## How I Want to Use It

- Read the event rules and scope before touching anything.
- Track each objective separately.
- Save screenshots only when they add evidence.
- Write down hints, failed ideas, and final reasoning.
- Practice writing a final summary like I would for a lab report.
- Keep writeups clearly marked by year because content changes.

## Objective Note Format

```text
Year:
Objective:
Status:
Location or challenge area:
Prompt:
Evidence collected:
Tools used:
Hint used:
What worked:
What failed:
Final answer or result:
Lesson:
```

## Evidence Habits

Useful evidence:

- challenge prompt
- relevant screenshot
- command output summary
- important file hash or name
- packet or log clue
- final reasoning

Not useful:

- huge screenshots with no explanation
- raw terminal dumps without notes
- spoilers copied from someone else
- screenshots that expose private account details

## Final Report Structure

A clean report could include:

1. Short overview.
2. Objective list.
3. Method for each objective.
4. Evidence for each objective.
5. Final answers or outcomes.
6. Lessons learned.
7. Appendix with commands or screenshots.

I should write in plain language. A future version of me should be able to read the report and understand the path without reopening every tab.

## Working With Hints

Hints are part of learning, but I should record them honestly.

When I use a hint, I should write:

- what I tried before the hint
- what the hint clarified
- what concept I was missing
- whether I could solve a similar task next time

This makes hints useful instead of invisible.

## Safety and Scope

- Stay inside the official challenge environment.
- Do not attack unrelated SANS systems or real external services.
- Do not publish sensitive account information, tokens, or private challenge state.
- Keep writeups clearly marked by year because challenge content changes.
- Follow the event's publication and participation rules.
